REPORTS-INVITATIONS-001 — Move Invitation Tracking to Reports

STEP 0 — SAFETY FIRST

Before making changes:

1. Create Replit checkpoint/snapshot.
2. Create git tag:

v3.9-before-invitation-report-migration

3. Confirm build passes.

--------------------------------------------------
PHASE 1 — ANALYSIS FIRST
--------------------------------------------------

Before removing anything:

Investigate whether Management → Invitations still contains any functionality that is NOT already available through:

Management → Members

Provide a report showing:

- Function
- Available in Members (Yes/No)
- Available elsewhere (Yes/No)

Examples:

- Create invitation
- Send invitation
- Resend invitation
- Revoke invitation
- Invitation status
- Invitation acceptance tracking
- Invitation expiry tracking
- Bulk operations
- Invitation statistics

Do NOT remove the menu yet.

--------------------------------------------------
PHASE 2 — DECISION
--------------------------------------------------

If ALL functionality already exists in Members:

Then:

- create another checkpoint/snapshot
- create git tag:

v3.9-before-remove-invitations-menu

- remove Management → Invitations menu

Keep all backend invitation functionality.

Only remove the redundant navigation/menu.
