# 1. Add rate limiting

- POST /api/auth/login

- POST /api/auth/verify-2fa
- POST /api/invitations/accept

# 2. Add TOTP brute-force protection

Protect 2FA verification attempts.

Requirements:

- Track failed TOTP attempts per user/session/IP.
- After 5 failed attempts, temporarily lock 2FA verification for 10–15 minutes.
- Log failed and locked 2FA attempts.
- Show a user-friendly error message.

# 3. Invalidate sessions after critical changes

Destroy active sessions for a user when:

- Password is changed
- Password is reset
- User is deactivated
- Administrator role is removed
- Site Administrator role is removed
- PM Super Administrator role is removed
- Protected domain access is removed

Do not allow old sessions to retain removed privileges.

# 4. Add password history protection

Implement password history.

Requirements:

- Store the last 5 password hashes.
- Prevent reuse of the last 5 passwords.
- Apply to password reset and password change.
- Do not store plain-text passwords.
