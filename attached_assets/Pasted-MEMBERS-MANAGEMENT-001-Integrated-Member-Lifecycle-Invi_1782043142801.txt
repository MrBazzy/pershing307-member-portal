MEMBERS-MANAGEMENT-001 — Integrated Member Lifecycle & Invitation Management

STEP 0 — SAFETY FIRST

Before making changes:

1. Create Replit checkpoint/snapshot.
2. Create git tag:

v3.9-before-member-management-redesign

3. Confirm build passes.

--------------------------------------------------
GOAL
--------------------------------------------------

Redesign Member Management so that:

- Members can be created before invitations are sent.
- Invitations become part of Member Management.
- Member details become easier to manage.
- Delete Member replaces Remove Test User.
- Invitation status becomes visible and traceable.
- Member history becomes visible through a timeline.

--------------------------------------------------
CURRENT PROBLEM
--------------------------------------------------

The current member side panel is becoming too small.

The portal now contains:

- member information
- invitation information
- roles
- degrees
- security information
- onboarding status
- member history

This no longer fits logically in a small sidebar.

--------------------------------------------------
NEW MEMBER DETAIL LAYOUT
--------------------------------------------------

Replace the current member sidebar with:

Management
→ Members
→ [Member Name]

Use a dedicated Member Detail page or a large responsive drawer.

SECTION 4 — INVITATIONS
--------------------------------------------------

Integrate Invitations into Member Management.

Do NOT remove the invitation system.

Keep:
- invitation tokens
- invitation links
- invitation validation

Move operations into the member page.

Buttons:

Create Invitation
Send Invitation
Resend Invitation
Revoke Invitation

Show:

- invitation status badge
- creation date
- sent date
- acceptance date
- expiry date

--------------------------------------------------
SECTION 5 — CREATE MEMBER WITHOUT INVITATION
--------------------------------------------------

Add:

Create Member

Administrator can create a member profile without sending an invitation.

Status:

Not Invited

This allows:

- preparing members in advance
- importing members
- creating test users
- completing information before onboarding

No email should be sent automatically.

Invitation can be sent later.

Show:

- Passkey status

- 2FA status

Allow revocation if already supported.

--------------------------------------------------
SECTION 7 — DANGER ZONE
--------------------------------------------------

Replace:

Remove Test User

with:

Delete Member

Delete Member becomes the single supported delete action.

--------------------------------------------------
DELETE MEMBER BEHAVIOR
--------------------------------------------------

Show warning:

You are about to delete a member account.

This action may remove account access and associated member records.

Are you absolutely sure?

Require confirmation.

Administrator must type:

First Name + Last Name

Example:

Type:
John Smith

before Delete button becomes active.

--------------------------------------------------
DELETE RESTRICTIONS
--------------------------------------------------

Do NOT allow deletion of:

- bootstrap administrator
- currently logged-in user

Site Administrator may NOT delete:

- PM Super Administrator

Only PM Super Administrator may delete another PM Super Administrator.

--------------------------------------------------
TEST MEMBERS
--------------------------------------------------

If Test Member functionality exists:

Keep:

Test Member badge

But remove:

Remove Test User

Delete Member now handles both normal and test accounts.
