* Folder structure
* Access control

* Audit logging

* File uploads
* Downloads
* Search
* Versioning
* Drag & Drop
* Document preview

These will be implemented in a later sprint.

## FOLDER VISIBILITY

Important:

If a user has no access:

The folder must be completely hidden.

Do NOT show the folder and display "Access Denied".

Users should only see folders they are allowed to access.

---

## SUBFOLDERS

Site Administrator and PM Super Administrator must be able to:

* Create subfolder
* Rename subfolder
* Delete empty subfolder

Subfolders inherit the permissions of the parent folder.

Example:

Treasury Documents
├ Budgets
├ Annual Accounts
├ Bank Statements

All inherit Treasury permissions.

No custom subfolder permissions in this sprint.

---

## FOLDER DESCRIPTION

Each folder may have:

* Title
* Description

Example:

Meeting Minutes

Description:
Official approved minutes of Lodge meetings.

Show description when opening folder.

## AUDIT LOGGING

Log:

* Folder created
* Folder renamed
* Folder deleted
* Subfolder created
* Subfolder renamed
* Subfolder deleted

Use full names.

Example:

John Smith created subfolder "2026" in Meeting Minutes.
